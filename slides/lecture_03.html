01 · Introduction
Operators and Expressions
📖 Key Definitions
An operator is a symbol that tells Python what kind of calculation to perform.

The numbers or variables used are called operands.

An expression is a combination of values, variables, and operators that Python evaluates to produce a value.
In mathematics you write 5 + 3 or 10 / 2. Python understands exactly the same expressions.
📝 Analogy
Think of an operator as a verb in a sentence. Just as the verb "add" tells you what action to perform, the operator + tells Python to perform addition.

02 · Floor Division
Floor Division — //
📖 Definition
Floor division divides two numbers and rounds the result down to the nearest whole number. The result is always an integer (when both operands are integers).
Python
chocolates = 17
people = 5

each_gets = chocolates // people
print(each_gets)
Output
3
17 // 5 — how many times does 5 fit completely into 17? Answer: 3. The remainder is discarded.
🍫 Analogy: Chocolates
You have 17 chocolates and 5 people.

Each person gets 3 chocolates (floor division).
2 are left over (modulus — next slide).
17 chocolates ÷ 5 people
🍫🍫🍫person 1
🍫🍫🍫person 2
...same for persons 3, 4, 5
→ each person gets 3 chocolates

02 · Modulus
Modulus (Remainder) — %
📖 Definition
The modulus operator % returns the remainder after division. If 17 = (5 × 3) + 2, then 17 % 5 = 2.
Python
leftover = 17 % 5
print(leftover)  # 2 chocolates left
Output
2
Common Use: Even or Odd
Python
number = 14
remainder = number % 2
print(remainder)  # 0 = even
Output
0
🍫 Back to the Chocolates
17 chocolates, 5 people.
Each gets 3 (floor div: 17 // 5 = 3).
5 × 3 = 15 chocolates given out.
17 − 15 = 2 left over.

17 % 5 gives you that 2.
number % 2 == 0
The number is even
number % 2 == 1
The number is odd

Exponentiation — **
📖 Definition
The ** operator raises a number to a power. 2 ** 8 means 2 × 2 × 2 × 2 × 2 × 2 × 2 × 2.

# Square root (power of 0.5)

print(root)

5.0

Raising a number to the power of 0.5 is equivalent to taking its square root.
25 ** 0.5 → 5.0

Expressions in Python
Combining values, variables, and operators to produce a result
04 · Expressions
What Is an Expression?
📖 Definition
An expression is a combination of values, variables, and operators that Python can evaluate to produce a result.
Valid Expressions
Python
5 + 3         # evaluates to 8
10 * 2 - 4    # evaluates to 16
100 / 4 + 5   # evaluates to 30.0
Assigning an Expression to a Variable
Python
total = 10 * 2 - 4
print(total)
Output
16
Python first evaluates the expression 10 * 2 - 4 to get 16, then stores that value in the variable total.
📝 The Pattern
variable = expression

1. Python evaluates the right-hand side
2. The result is stored in the variable
3. You can use the variable later
💡 Tip
Use meaningful variable names. total_cost = price * quantity is much clearer than x = a * b.

Which operation does Python perform first?
05 · Operator Precedence
Order of Operations
When an expression has more than one operator, Python follows operator precedence — a set of rules determining which operation is performed first.
1st — Highest ** Exponentiation
2nd *  /  //  % Multiply, Divide, Floor, Mod
3rd — Lowest +  − Add, Subtract
💡 Same Precedence → Left to Right
When two operators have the same precedence, Python evaluates from left to right.
📝 Analogy: Recipe Rules
You don't follow a recipe ingredient by ingredient — you follow the order: prepare first, mix, then bake. Python follows the same idea with operators.
✅ Override with Parentheses ()
You can always use () to control evaluation order explicitly and make your intent clear.

2 + 3 * 4 → 14
(2 + 3) * 4 → 20

06 · Variables in Calculations
Using Variables to Store and Reuse Values
Variables make expressions more readable and allow values to be reused. Instead of repeating the same number, store it in a variable and use the name.
Python
price_per_kg = 2.500
quantity_kg = 4

total_cost = price_per_kg * quantity_kg
print("Total cost:", total_cost)
Output
Total cost: 10.0
The expression price_per_kg * quantity_kg is easy to understand at a glance. If the price changes, you only update one variable.
📖
Readability
price * qty is clearer than 2.5 * 4
♻️
Reusability
Use a value many times without retyping it
✏️
Easy to Modify
Change the value once — it updates everywhere
🐛
Fewer Errors
No risk of typos when reusing a value

Mini-Program 2 — Average of Three Marks
📊 Average Marks
Python
mark1 = 78
mark2 = 85
mark3 = 92

total = mark1 + mark2 + mark3
average = total / 3

print("Total marks:", total)
print("Average mark:", average)

The three marks are summed to get the total, then divided by 3. Note the result is 85.0 (a float) because / always returns a float.

2
255 / 3→85.0(average — float!)

07 · Mini-Programs
Mini-Program 3 — Price After Discount
🏷️ Discount Calculator
Python
original_price = 120
discount_percent = 15

discount_amount = original_price * discount_percent / 100
final_price = original_price - discount_amount

print("Original price:", original_price)
print("Discount amount:", discount_amount)
print("Final price:", final_price)
Output
Original price: 120
Discount amount: 18.0
Final price: 102.0
The discount amount is calculated as a percentage of the original price. The final price is obtained by subtracting the discount.
1
120 * 15→1800(* left to right)
2
1800 / 100→18.0(/ same level)
3
120 - 18.0→102.0(final price)
💡 Precedence Note
* and / have the same precedence, so Python evaluates left to right: multiply first, then divide.
🎯 Try It
Change to discount_percent = 25 and original_price = 200. What is the final price?

Common Mistakes
Five errors beginners make — and how to fix each one
08 · Common Mistakes
Mistakes 1 & 2
❌ Mistake 1: Using / When Floor Division Is Needed
When you need a whole number (e.g. counting chapters), use // not /.
❌ Wrong
Gives decimal
chapters = 100 / 12
# 8.333333...
✅ Fixed
Fixed
chapters = 100 // 12
# 8 (complete chapters)
❌ Mistake 2: Confusing * and ** for Powers
* is multiplication. ** is exponentiation. These are different operations!
❌ Wrong (3²≠9)
Wrong operator
result = 3 * 3
# 9 (just 3×3)
✅ Fixed (3³=27)
Fixed
result = 3 ** 3
# 27 (3×3×3)
08 · Common Mistakes
Mistakes 3, 4 & 5
❌ Mistake 3: Ignoring Operator Precedence
Without parentheses, * is evaluated before +. Use () when you intend addition first.
❌ Gives 14
Wrong result
result = 2 + 3 * 4
# Expected 20, got 14
✅ Gives 20
Fixed
result = (2 + 3) * 4
# Correct: 20
❌ Mistake 4: Adding String + Number
If a value is a string ("50"), Python cannot add it to a number. Use an integer or convert with int().
❌
TypeError
price = "50"
total = price + 5
✅
Fixed
price = 50
total = price + 5
❌ Mistake 5: Division by Zero
Dividing any number by zero is undefined. Python raises a ZeroDivisionError. Always ensure the divisor is not zero.
❌
ZeroDivisionError
share = 100 / 0
# Crash!
✅
Fixed
share = 100 / 4
# 25.0
Summary
Lecture 3 — Key Concepts
Key Operators
  • + adds · - subtracts · * multiplies
  • / divides — always returns a float
  • // divides — returns a whole number (floor)
  • % returns the remainder after division
  • ** raises a number to a power
Key Concepts
  • An expression is a combination of values, variables, and operators
  • Precedence order: ** → * / // % → + -
  • Use () to control evaluation order explicitly
  • Store values in variables for readable, reusable code
  • Check data types — mixing strings and numbers causes errors
