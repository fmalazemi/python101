Section 01
What Is Programming?
Understanding instructions, programs, and how computers think.
Section 01
What Is a Program?
  • A program is a set of instructions that tells a computer what to do.
  • Writing those instructions is called programming (or coding).
  • A computer does exactly what you tell it — nothing more, nothing less.
  • If your instructions have an error, the computer will not guess what you meant.
A program must be precise, ordered, and unambiguous.
⚡
Extremely Fast
Computers execute billions of instructions per second.
🔒
Completely Literal
It follows your exact instructions — errors cause failures.
Section 01 · Analogy
🍵 A Program Is Like a Recipe

A recipe gives clear, ordered steps. A program works the same way.

Recipe: Making Tea
1
Boil water.
2
Place a tea bag in a cup.
3
Pour hot water into the cup.
4
Wait three minutes.
5
Remove the tea bag.
Key Idea
Each step is clear, ordered, and unambiguous. A recipe does not say "do something with water." It gives exact instructions.
A program: Each instruction must be precise and written in the correct order — just like a recipe.

❌ Wrong: "sat cat the mat on the"

Section 04
Writing Your First Program
Using the print() function to display output
Section 04
The print() Function
  • The most fundamental action in Python: displaying information on screen.
  • A function is a named instruction that performs a specific task.
  • print() is a built-in Python function — it is already available for you to use.
Syntax
Python
print(something)
print
The name of the function. Always lowercase.
( )
Parentheses tell Python you are calling this function. Always required.
something
The value you want to display goes inside the parentheses.

Displaying Text (Strings)
Python

▶ Output
Hello, world!
🔑 Key rule: Text must be enclosed in quotation marks — these are called strings.
How it works
1
print — name of the function
2
() — tells Python to call the function
3
"Hello, world!" — the string to display
4
Python displays the text on screen

Section 05
Common Mistakes
The most frequent beginner errors — and how to fix them.
Section 05 · Mistakes 1 & 2
⚠️ Missing Quotes & Missing Parentheses
Mistake 1: No quotes around text
Python
# ❌ Wrong
print(Hello)
✖ Error
NameError: name 'Hello' is not defined
print("Hello")  # ✅ Correct
Mistake 2: No parentheses
Python
# ❌ Wrong
print "Hello"
✖ Error
SyntaxError: Missing parentheses in call to 'print'
print("Hello")  # ✅ Correct
Section 05 · Mistakes 3 & 4
⚠️ Mismatched Quotes & Wrong Capitalisation
Mistake 3: Mismatched quotes
Python
# ❌ Wrong
print("Hello')
✖ Error
SyntaxError: EOL while scanning string literal
Use matching quotes: "Hello" or 'Hello'
Mistake 4: Capital P in Print
Python
# ❌ Wrong
Print("Hello")
✖ Error
NameError: name 'Print' is not defined
Python is case-sensitive. Always use lowercase: print
Section 05 · Mistake 5
⚠️ Numbers in Quotes vs. Real Numbers
❌ Numbers as Text
print("5" + "3")
▶ Output
53

Joins the text — this is called concatenation!

✅ Real Numbers
print(5 + 3)
▶ Output
8

Adds the numbers — correct math!

Rule: Numbers in quotation marks are treated as text, not as numbers.

If you want to do math, do NOT put numbers in quotes.
String
"5" + "3" = "53" (joined text)
Integer
5 + 3 = 8 (real math)

Summary
✅ Core Rules to Remember
  • Text (strings) must be enclosed in quotation marks: "like this" or 'like this'.
  • Numbers do NOT use quotation marks: 42 or 3.14.
  • print must always be written in lowercase.
  • Always include parentheses after print: print().
  • Python runs your program from top to bottom, one line at a time.
These five rules will help you avoid the most common beginner mistakes. Return to them whenever you are unsure.
