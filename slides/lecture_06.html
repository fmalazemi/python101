Motivation
Why Repetition Matters
Repetition is one of the most powerful ideas in programming. Without loops, a program would need to repeat the same lines of code manually — once per repetition. This is impractical and error-prone.
🛒
Cashier
Scans items one by one until there are no more items left.
📖
Student Practising
Repeats a vocabulary word until they can spell it correctly.
🏧
ATM PIN Entry
Asks for a PIN again and again until the correct PIN is entered.
💡 The Solution: while Loop
The while loop allows a program to repeat a block of code automatically, as many times as needed, based on a condition. Mastering loops is an essential step toward writing useful, real-world programs.

01 · Introduction
From if to while — Repeating Actions
📖 Definition
A loop is a programming structure that repeats a block of code.

The while loop repeats a block of code as long as a condition remains true. When the condition becomes false, the loop stops.
if statement
Runs the block once — only if the condition is true.
while loop
Runs the block repeatedly — as long as the condition is true.
🚪 Analogy: Security Guard
The guard checks a rule: "Is this person on the list?"

As long as the answer is "No", the guard keeps turning people away.

The moment the answer is "Yes", the guard stops checking and lets the person in.

The loop keeps running while the condition is true and stops when it becomes false.

    01 · Structure
    Why the Update Step Is Critical
    The update step is a statement inside the loop body that changes the variable used in the condition. Without it, the condition never changes — and the loop runs forever.
    ⚠️
    Without Update
    Condition never changes → loop runs forever → infinite loop!
    ✅
    With Update
    Condition moves toward False → loop ends correctly.
    📝 Common Update Patterns
    counter = counter + 1  (count up)
    counter = counter - 1  (count down)
    counter += 1             (shorthand)
    🪜 Analogy: Climbing Stairs
    The condition is: "Are there more steps?"
    The update step is: "Take one step."

    Each time you take a step, you get closer to the top. Without taking a step, you would be stuck at the same stair forever — an infinite loop.
    ⚠️ Wrong Direction Is Also Dangerous
    If your update moves the variable away from making the condition False (e.g. you want to count up to 5 but you write counter - 1), the loop also runs forever.

    04
    Repeating Until Correct Input
    Input validation — keep asking until the user provides an acceptable answer
    04 · Input Validation
    Example 4 — Keep Asking for a Positive Number
    Python
    number = int(input("Enter a positive number: "))

while number <= 0:
    print("That is not a positive number. Try again.")
    number = int(input("Enter a positive number: "))

print("Thank you! You entered:", number)
    Example Interaction
    Enter a positive number: -3
That is not a positive number. Try again.
Enter a positive number: 0
That is not a positive number. Try again.
Enter a positive number: 7
Thank you! You entered: 7
    The program asks for input once before the loop (to get the first value), and once inside the loop (to get the next value if the first was invalid).
    💡 The Pattern — Input Validation Loop
    1. Ask for input before the loop
    2. Check input in the while condition
    3. If invalid, show a message and ask again inside the loop
    4. When input is valid, condition becomes False → loop ends
    🎯 Try It
    Modify to ask for a number between 1 and 10. The loop should repeat until the user enters a valid number in that range.
    04 · Input Validation
    Example 5 — Password Check Loop
    Python
    password = input("Enter the password: ")

while password != "python123":
    print("Incorrect password. Try again.")
    password = input("Enter the password: ")

print("Access granted.")
    Example Interaction
    Enter the password: hello
Incorrect password. Try again.
Enter the password: python
Incorrect password. Try again.
Enter the password: python123
Access granted.
    The condition password != "python123" is True as long as the entered password is wrong. The loop ends only when the correct password is entered.
    ⚠️ Case Sensitivity
    String comparison is case-sensitive. "Python123" would not be accepted — it must match exactly.
    💡 Why != Here Instead of <= ?
    For strings, we compare using == or !=. The loop continues while the password is NOT equal to the correct one.

    05
    Infinite Loops
    When the condition never becomes False — and how to avoid it
    05 · Infinite Loops
    What Is an Infinite Loop?
    ⚠️
    An infinite loop is a loop that never stops because its condition never becomes False. This is one of the most common beginner mistakes.
    Cause 1 — Missing Update Step
    ⚠️ Do NOT run — infinite loop
    counter = 1
while counter <= 5:
    print(counter)
    # update step is MISSING!
# counter never changes → always True
    Cause 2 — Update Goes Wrong Direction
    ⚠️ Do NOT run — infinite loop
    counter = 1
while counter <= 5:
    print(counter)
    counter = counter - 1  # going DOWN!
# counter: 0, -1, -2... always < 5
    🛁 Analogy: Bathtub
    The condition is: "Is there still water?"
    The update step is: "Pull the plug."

    If you never pull the plug (missing update), or keep adding water (wrong direction), the tub never empties — you are stuck in an infinite loop.
    🛑 How to Stop an Infinite Loop
    If you accidentally run an infinite loop, stop the program by pressing Ctrl + C in the terminal.
    05 · Infinite Loops
    How to Avoid Infinite Loops — 3 Questions to Ask
    Before running any while loop, always ask yourself these three questions:
    1
    Does the loop have a condition? The condition must be a Boolean expression that can evaluate to True or False.
    2
    Does the loop body change the variable used in the condition? There must be an update step somewhere inside the loop body.
    3
    Does the update step move toward making the condition False? The variable must change in the correct direction — towards the boundary that makes the condition False.
    ⚠️ If You Answer "No" to Any Question
    You have an infinite loop. Fix the problem before running the code. Always trace through your loop mentally for a few iterations to confirm it will eventually stop.
    06
    Common Mistakes
    Five errors beginners make — and how to fix each one
    06 · Common Mistakes
    Mistakes 1 & 2
    ❌ Mistake 1: Forgetting the Update Step
    Without count = count + 1, the variable stays at 1 forever → infinite loop.
    ❌ Infinite Loop
    Runs forever
    count = 1
while count <= 3:
    print("Hello")
    # no update!
    ✅ Fixed
    Python
    count = 1
while count <= 3:
    print("Hello")
    count = count + 1
    ❌ Mistake 2: Wrong Initial Value — Loop Never Runs
    If the condition is already False before the loop starts, the loop body never executes.
    ❌ Never Runs
    Prints nothing
    count = 10
while count <= 5:
    print(count)
    count = count + 1
# 10 <= 5 is False → skip
    ✅ Fixed
    Python
    count = 1   # start inside range
while count <= 5:
    print(count)
    count = count + 1
    06 · Common Mistakes
    Mistakes 3, 4 & 5
    ❌ Mistake 3: Off-by-One Error
    Using < instead of <= stops the loop one iteration too early. Always check which comparison operator you need.
    ❌ Prints 1–4
    Stops too early
    while count < 5:
    print(count)
    count = count + 1
    ✅ Prints 1–5
    Fixed
    while count <= 5:
    print(count)
    count = count + 1
    ❌ Mistake 5: Using = Instead of == in Condition
    A single = is assignment. The comparison operator is ==. Using = in a condition causes a SyntaxError.
    ❌ SyntaxError
    SyntaxError
    while answer = "yes":
    print("You said yes.")
    ✅ Fixed
    Fixed
    while answer != "yes":
    answer = input("Type yes: ")
    ❌ Mistake 4: Update Step Outside the Loop Body
    If the update step is not indented, it is outside the loop. The loop runs the body, but never updates — causing an infinite loop. The update runs only once, after the loop ends (which is too late).
    ❌ Update is Outside
    Infinite Loop
    while count <= 3:
    print(count)
count = count + 1  # not indented!
    ✅ Update is Inside
    Fixed
    while count <= 3:
    print(count)
    count = count + 1  # indented!
    Summary
    Lecture 6 — Key Concepts
    • A while loop repeats a block of code as long as a condition is True
    • Every loop has three parts: condition, body, and update step
    • The condition is checked before each iteration — if False from the start, the body never runs
    • An infinite loop occurs when the condition never becomes False — always include an update step
    • while loops are used for counting, accumulating, and input validation
    • Tracing — recording variable values at each iteration — is the best way to understand a loop
    • Use == for comparison, = for assignment — never use = in a condition
