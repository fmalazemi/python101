01 · Introduction
What Is a Function?
📖 Definition
A function is a named block of code that performs a specific task. You define it once and can execute it ("call" it) at any point in your program.
  • The input to a function is called a parameter (or argument)
  • The output from a function is called the return value
  • Some functions take no input and produce no output — they simply perform an action
🏭 Analogy: A Factory Machine
Think of a function as a machine in a factory. You place raw materials in (input), the machine performs an operation, and produces a finished product (output).

You can use the same machine as many times as you need — you do not rebuild it each time.

01 · Defining Functions
Anatomy of a Function Definition
defkeyword greetname(nameparameter):
message = "Hello, " + namebody
returnreturn message
  • def — keyword that begins the definition
  • Function name — follows variable naming rules
  • () — holds parameters (can be empty)
  • : — colon ends the function header
  • Indented body — code that runs when called
⚠️ Define ≠ Run
Defining a function does not run it. You must call the function separately. The body only executes when called.
Python
def greet():
    print("Hello! Welcome.")

# Nothing runs yet...

greet()   # ← This runs it!
Output
Hello! Welcome.

Calling a Function — Reusability in Action
💡 The Power of Reuse
Call the same function as many times as needed. Change the separator in one place — updated everywhere.
Python
def print_separator():

--------------------
Grade: A

Instead of writing print("----") three times, define once, call whenever needed.
❌ Without Functions
Repeated code everywhere. Changing separator = find and edit every line.
✅ With Functions

02 · Parameters
Functions with One Parameter
📖 Parameter vs Argument
A parameter is the variable placeholder in the function definition.
An argument is the actual value you pass when calling.
Python
def greet_student(name):     # ← parameter
    print("Hello,", name, "!")

greet_student("Sara")        # ← argument
greet_student("Omar")
greet_student("Lena")
Output
Hello, Sara !
Hello, Omar !
Hello, Lena !
What Python does on each call
greet_student("Sara")
→
name = "Sara"
greet_student("Omar")
→
name = "Omar"
greet_student("Lena")
→
name = "Lena"
💡 Same Function, Different Results
Each call assigns a new value to name. The function body runs with the new value.

02 · Parameters
Functions with Multiple Parameters
📖 Multiple Parameters
Separate parameters with commas. The order of arguments must match the order of parameters.
Python
def calculate_area(length, width):
    area = length * width
    print("The area is:", area)

calculate_area(5, 3)
calculate_area(10, 4)
Output
The area is: 15
The area is: 40
Argument order matters
calculate_area(5, 3)
→
length=5, width=3
calculate_area(10, 4)
→
length=10, width=4
⚠️ Order Matters!
First argument → first parameter, second → second. Wrong order gives wrong results with no error message.

Write calculate_perimeter(length, width). Formula: 2 * length + 2 * width

Return Values
return · send a value back to the caller
03 · Return Values
Returning Values from a Function
📖 The return Keyword
A return value is the output that a function sends back to the calling code. Use return to send it. Once Python reaches return, it immediately exits the function.
Python
def calculate_area(length, width):
    area = length * width
    return area          # ← sends value back

room_area = calculate_area(6, 4)
print("Room area:", room_area)

print("Area:", calculate_area(7, 3))
Output
Room area: 24
Area: 21
Call: calculate_area(6, 4)
↓ enters function
Inside: area = 6 × 4 = 24 → return 24
↓ value sent back
Caller: room_area = 24 ✓
✅ Why return?
The returned value can be stored, printed, or used in further calculations — unlike a value that is only printed inside the function.
03 · Return Values
Example — Maximum of Two Numbers
Python
def find_maximum(a, b):
    if a > b:
        return a     # exits function immediately
    else:
        return b

result = find_maximum(15, 9)
print("The maximum is:", result)

print("The maximum is:", find_maximum(100, 250))
Output
The maximum is: 15
The maximum is: 250
A function can have more than one return statement, but only one executes per call. As soon as Python hits a return, it exits the function.
Call 1: find_maximum(15, 9)
15 > 9 → True → return 15
Call 2: find_maximum(100, 250)
100 > 250 → False → else → return 250
🎯 Try It
Write find_minimum(a, b) that returns the smaller of two numbers.

print() vs return
The most common source of confusion for beginners
04 · print vs return
The Difference Between print() and return
print() inside a functionreturn inside a function
What it doesDisplays output on the screenSends a value back to the caller
Use result later?No — value is displayed and goneYes — caller can store or reuse it
Visible to user?Yes (immediately)Only if you print the returned value
If not usedFunction returns NoneFunction returns the specified value
💡 Rule of Thumb
Use return when you want to use the result elsewhere — store it, calculate with it, pass it on. Use print() only to display something and you do not need the value again.
04 · print vs return
Side-by-Side Comparison
⚠️ Version with print()
Python
def add_with_print(a, b):
    print(a + b)   # displays 7

result1 = add_with_print(3, 4)
print("Result1:", result1)
Output
7
Result1: None
The function prints 7 but returns nothing. result1 is assigned None — Python's way of saying "nothing was returned."
✅ Version with return
Python
def add_with_return(a, b):
    return a + b   # sends 7 back

result2 = add_with_return(3, 4)
print("Result2:", result2)
Output
Result2: 7
The function returns 7. The calling code stores it in result2 and can use it freely.

Common Mistakes
Five errors beginners make — and how to fix each one
05 · Common Mistakes
Mistakes 1 & 2
❌ Mistake 1: Defining But Not Calling
Defining a function does nothing. You must call it to make it run.
❌ No Output
Nothing happens
def greet():
    print("Hello!")
# Never called!
✅ Fixed
Fixed
def greet():
    print("Hello!")
greet()  # ← runs it
❌ Mistake 2: Calling Before Defining
Python reads top to bottom. The function must exist before you call it.
❌ NameError
NameError
greet()  # Error! Not defined yet.

def greet():
    print("Hello!")
✅ Fixed
Fixed
def greet():
    print("Hello!")
greet()  # ← after definition
05 · Common Mistakes
Mistakes 3, 4 & 5
❌ Mistake 3: print() Instead of return
Using print() but trying to use the result gives None → TypeError.
❌ TypeError
None + 1 → Error
def double(n):
    print(n * 2)  # None!
result = double(5)
print(result + 1) # Crash!
✅ Fixed
Fixed
def double(n):
    return n * 2  # value!
result = double(5)
print(result + 1) # 11 ✓
❌ Mistake 4: Wrong Number of Arguments
The count of arguments must exactly match the count of parameters.
❌ TypeError
TypeError
def greet_student(name):
    print("Hello,", name)
greet_student("Ali", "Omar")
# 2 args, 1 param → Error!
✅ Fixed
Fixed
greet_student("Ali")
# 1 arg, 1 param ✓
❌ Mistake 5: Using a Variable Outside Its Function (Scope)
Variables inside a function exist only within it. To use a value outside, return it.
❌ NameError
NameError
def calculate():
    answer = 42
calculate()
print(answer)  # answer is gone!
✅ Fixed
Fixed
def calculate():
    answer = 42
    return answer
result = calculate()
print(result)  # 42 ✓

def greet():
    print("Hello!")

# With parameter
def greet(name):
    print("Hello,", name)

    return n * n

def add(a, b):
    return a + b

# Calling

greet("Sara")
