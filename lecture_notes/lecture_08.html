01 · Strings
What Is a String?
📖 Definition
A string is a sequence of characters. A character can be a letter, digit, space, or symbol. In Python, a string is written between quotation marks.
Python
greeting = "Hello"
name = 'Sara'
message = "Welcome to Python!"

print(greeting)
print(name)
Output
Hello
Sara
You can use either single quotes '...' or double quotes "..." — both work the same way.
🎬 Analogy: Cinema Seats
Think of a string as a row of seats. Each seat holds one character. Each seat has a number — called an index — that tells you its position.

The first seat is numbered 0, the second is 1, and so on.

01 · Indexing
Positive Indexing — Zero-Based Counting
📖 Zero-Based Indexing
Python counts from 0, not 1. The first character is at index 0, the second at index 1, and so on. Access a character by writing the variable name followed by [index].
Python
language = "Python"
print(language[0])   # P
print(language[1])   # y
print(language[5])   # n
Output
P
y
n

⚠️ First Index is 0, Not 1
This is called zero-based indexing. It is one of the most important rules in Python — and a very common source of off-by-one errors.
01 · Indexing
Negative Indexing — Counting from the End
📖 Negative Indices
Python allows negative indices that count from the end of the string. Index -1 is the last character, -2 is second to last, and so on.
Python
word = "Python"
print(word[-1])   # n  (last)
print(word[-2])   # o  (second to last)
print(word[-6])   # P  (first)
Output
n
o
P
✅ When to Use Negative Indexing
Useful when you want to access characters at the end of a string without knowing its exact length — e.g. always getting the last character.

02 · Length
The len() Function
📖 Definition
The len() function returns the number of characters in a string — including letters, digits, spaces, and symbols.
Python
name = "Ahmed"
print(len(name))         # 5

sentence = "I love Python"
print(len(sentence))     # 13 — spaces count!
Output
5
13
⚠️ Spaces Are Characters Too!
"I love Python" has 13 characters — the 2 spaces are counted. Every character in the string is included in the length.
"Ahmed" — 5 characters:
0
A
1
h
2
m
3
e
4
d
len("Ahmed") = 5
💡 len() vs .len()
len() is a function, not a method. Call it with the variable inside the parentheses:
len(name) ✅
name.len() ❌ — this causes an error!

02 · Concatenation
String Concatenation — Joining with +
📖 Definition
Concatenation means joining two or more strings together. Use the + operator.
Python
first_name = "Sara"
last_name = "Ali"
full_name = first_name + " " + last_name
print(full_name)   # Sara Ali

greeting = "Hello, "
name = "Omar"
message = greeting + name + "!"
print(message)     # Hello, Omar!
Output
Sara Ali
Hello, Omar!
🪢 Analogy: Connecting Rope
Think of concatenation like connecting two pieces of rope. Each piece is a string, and + ties them together into one longer string.
"Sara"
+
" "
+
"Ali"
→
"Sara Ali"
⚠️ Don't Forget the Space!
Without the space string " ", the result would be "SaraAli". You must explicitly add spaces between words.
02 · Repetition
String Repetition — Multiplying with *
📖 Definition
You can repeat a string a certain number of times using the * operator. The result is one string containing the repeated content.
Python
line = "-" * 20
print(line)   # --------------------

word = "ha"
laugh = word * 3
print(laugh)  # hahaha

print("Python " * 3)  # Python Python Python
Output
--------------------
hahaha
Python Python Python
Practical Uses of Repetition
Visual Separators
"-" * 20
Creates clean divider lines in output.
Pattern Generation
"*" * i
Grows a pattern inside a loop.
Emphasis
"!" * 5
Adds emphasis to messages.
Report Formatting
"=" * 30
Creates headers for text reports.

String Methods
upper() · lower() · strip() · replace()
03 · Methods
What Is a String Method?
📖 Definition
A method is a built-in action that belongs to a specific data type. String methods perform operations on string values.

Call a method by writing: variable.method()
Python
text = "hello"
print(text.upper())   # HELLO
print(text.lower())   # hello (unchanged)
Output
HELLO
hello
The Four Methods We Study
.upper()case
Converts all characters to UPPERCASE
.lower()case
Converts all characters to lowercase
.strip()whitespace
Removes leading and trailing spaces
.replace(old, new)text
Replaces all occurrences of one substring with another
03 · Methods
upper() and lower()
Python
name = "sara"
print(name.upper())   # SARA

title = "INTRODUCTION TO PYTHON"
print(title.lower())  # introduction to python
Output
SARA
introduction to python
These methods do not change the original variable. They return a new string. The value of name remains "sara" after calling .upper().
🎯 Common Use Case: Comparing User Input
When comparing user input, convert everything to lowercase so that "YES", "Yes", and "yes" are treated as the same.
Python
answer = input("Continue? ").lower()

if answer == "yes":
    print("Continuing...")
This works whether the user types "YES", "Yes", or "yes" — all become "yes" after .lower().
03 · Methods
strip() — Removing Extra Whitespace
📖 What It Does
strip() removes leading and trailing whitespace — spaces, tabs, and newlines at the beginning and end of a string. It does not affect spaces in the middle.
Python
user_input = "   Ahmed   "
clean_input = user_input.strip()

print(clean_input)          # Ahmed
print(len(user_input))      # 11 (spaces counted!)
print(len(clean_input))     # 5
Output
Ahmed
11
5
✂️ Analogy: Trimming Paper
Think of strip() as trimming the extra blank space from the left and right edges of a piece of paper. The content in the middle stays the same.
Before: Ahmed
After: Ahmed
✅ Essential for Input Processing
Users often accidentally add spaces when typing. Always .strip() input before comparing or storing it to avoid silent bugs.
03 · Methods
replace(old, new) — Substituting Text
📖 Syntax
string.replace(old, new)
Replaces all occurrences of old with new. Case-sensitive.
Python
sentence = "I like cats. Cats are great."
new_sentence = sentence.replace("cats", "dogs")
print(new_sentence)
# I like dogs. Cats are great.

message = "Hello World"
print(message.replace("World", "Python"))
# Hello Python
Output
I like dogs. Cats are great.
Hello Python
⚠️ Case-Sensitive!
Only "cats" (lowercase) was replaced. "Cats" (capital C) was not changed because replace() treats uppercase and lowercase as different.
🎯 Practical Uses
  • Fixing common spelling mistakes in text
  • Sanitising input (e.g. remove special characters)
  • Building template messages with placeholders
  • Formatting data for display
🎯 Try It
Store "The sky is grey". Use replace() to change "grey" to "blue".

Strings in Python are immutable. Methods like upper() and replace() return a new string. To keep the result, assign it back: name = name.upper()

04 · Immutability
Strings Are Immutable
📖 What Is Immutability?
In Python, strings cannot be changed after they are created. This is called immutability. Methods like upper() and replace() do not modify the original string — they return a new one.
❌ Wrong — result is discarded
Python
name = "sara"
name.upper()       # result thrown away!
print(name)        # sara  ← unchanged
✅ Correct — save the result
Python
name = "sara"
name = name.upper()  # save the new string
print(name)          # SARA
What actually happens
name = "sara"→"sara"(original)
name.upper()→"SARA"(new string, discarded)
print(name)→"sara"unchanged!
⚠️ This Is the Most Forgotten Rule
If you call a method without assigning the result, the transformation is lost silently. There is no error message — the variable just stays unchanged.
✅ Rule to Remember
Always assign the result:
text = text.strip()
name = name.upper()
05
Common Mistakes
Four errors beginners make — and how to fix each one
05 · Common Mistakes
Mistakes 1 & 2
❌ Mistake 1: Concatenating a String with an Integer
The + operator cannot join a string and an integer directly. Convert the integer to a string with str(), or use an f-string.
❌ TypeError
TypeError
age = 20
msg = "I am " + age + " years old."
✅ Fixed
Fixed
msg = "I am " + str(age) + " years old."
# or:
msg = f"I am {age} years old."
❌ Mistake 2: Index Out of Range
Accessing an index that does not exist causes an IndexError. Always make sure the index is within the valid range (0 to len−1).
❌ IndexError
IndexError
word = "Hi"   # only indices 0, 1
print(word[5])  # does not exist!
✅ Fixed
Fixed
word = "Hi"
print(word[0])   # H
print(word[-1])  # i (last char)
05 · Common Mistakes
Mistakes 3 & 4
❌ Mistake 3: Forgetting That Strings Are Immutable
Calling a method without assigning the result does nothing visible. No error — the variable just stays unchanged. Always save the result.
❌ Result Lost
Silently does nothing
name = "sara"
name.upper()     # result discarded
print(name)      # still "sara"!
✅ Fixed
Fixed
name = "sara"
name = name.upper()  # save it!
print(name)          # SARA
❌ Mistake 4: Calling len() as a Method
len() is a built-in function, not a method. You cannot call it with a dot. It must be called with the variable inside the parentheses.
❌ AttributeError
AttributeError
name = "Omar"
print(name.len())   # Error!
# str has no .len() method
✅ Fixed
Fixed
name = "Omar"
print(len(name))   # 4 ✓
Summary
Lecture 8 — Key Concepts
  • A string is a sequence of characters enclosed in quotation marks
  • Characters are accessed via indexing, starting from index 0
  • Negative indices count from the end: -1 = last character
  • len() returns the number of characters (including spaces)
  • + concatenates strings · * repeats a string
  • Methods: upper(), lower(), strip(), replace()
  • Strings are immutable — always assign the method result back to a variable
  • Cannot concatenate str with int — use str() or an f-string
Quick Reference
s = "  Hello, World!  "

# Indexing
s[0]           # ' ' (first char)
s[-1]          # ' ' (last char)

# Length
len(s)         # 18

# Concatenation & Repetition
"Hi " + "there"  # "Hi there"
"=" * 10         # "=========="

# Methods
s.strip()           # "Hello, World!"
s.lower()           # "  hello, world!  "
s.upper()           # "  HELLO, WORLD!  "
s.replace(",", ";") # "  Hello; World!  "

# Save method results!
s = s.strip().lower()
