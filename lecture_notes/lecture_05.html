01 · Foundation
The Concept of Truth — Boolean Values
Every condition in Python evaluates to one of two values:
True
The condition is satisfied
False
The condition is not satisfied
These are called Boolean values, named after the mathematician George Boole.
Python
age = 20
print(age >= 18)   # True
print(age < 10)    # False
Output
True
False
🔗 How Logical Operators Fit In
Logical operators work by combining these True and False values to produce a new single True or False result.
andBoth must be True
orAt least one True
notReverses the value
02
Logical Operator: and
True only when BOTH conditions are True
02 · and
The and Operator
📖 Rule
The and operator returns True only if both conditions are True. If either is False, the entire expression is False.
Truth Table
Condition ACondition BA and B
TrueTrueTrue ✓
TrueFalseFalse
FalseTrueFalse
FalseFalseFalse
🎫 Analogy: Security Checkpoint
You may enter an event only if you have a valid ticket AND your name is on the guest list.

If either condition fails, you cannot enter. Both must be satisfied.
True and True = True
True and False = False
False and False = False
02 · and
Example — Age and Membership Check
Python
age = 25
is_member = True

if age >= 18 and is_member:
    print("Access granted.")
else:
    print("Access denied.")
Output
Access granted.
1
age >= 18→True(25 ≥ 18)
2
is_member→True(already True)
3
True and True→True→ if block runs
Because both conditions are True, the and expression evaluates to True.
⚠️ Try Changing the Values
Set is_member = False. Now one condition is False → and is False → "Access denied."
🎯 Try It Yourself
Change age to 15. What happens?
Add a third condition: age <= 60. What if age = 70?

Logical Operator: or
True when AT LEAST ONE condition is True
03 · or
The or Operator
📖 Rule
The or operator returns True if at least one condition is True. It is False only when both conditions are False.
Truth Table
Condition ACondition BA or B
TrueTrueTrue
TrueFalseTrue
FalseTrueTrue
FalseFalseFalse ✗
🏛️ Analogy: Museum Discount
A museum offers free admission to visitors who are students OR senior citizens.

You only need to satisfy one condition. You don't have to be both.
False or True = True
False or False = False

Logical Operator: not
Reverses the truth value of a condition
04 · not
The not Operator
📖 Rule
The not operator reverses the truth value. True becomes False. False becomes True.
Truth Table
Condition Anot A
TrueFalse
FalseTrue
Python
is_logged_in = False

if not is_logged_in:
    print("Please log in to continue.")
else:
    print("Welcome back!")
Output
Please log in to continue.
🚪 Analogy: Locked Door
A sign reads: "Entry allowed if the door is not locked."

If locked = True → not locked = False → no entry.
If locked = False → not locked = True → enter!
1
is_logged_in→False
2
not False→True(reversed!)
3
if True:→if block runs
🎯 Try It
Change is_logged_in to True. Now not True = False → "Welcome back!" prints instead.

Combining Multiple Conditions
Using and, or, not together in one expression
05 · Combining Conditions
Logical Operator Precedence & Combining
When combining and, or, and not in one expression, Python evaluates them in this order:
1st — Highest not Reversal
2nd and Both must be True
3rd — Lowest or At least one True
✅ Best Practice
Use parentheses () to make the intended meaning clear, even when not strictly required. Clarity prevents bugs.
Example — or with permission
Python
age = 17
has_permission = True

if age >= 18 or has_permission:
    print("You may proceed.")
else:
    print("Access not allowed.")
Output
You may proceed.
1
age >= 18→False(17 < 18)
2
has_permission→True
3
False or True→True
05 · Combining Conditions
Example — Using and with not
Python
temperature = 35
is_raining = False

if temperature > 30 and not is_raining:
    print("Good weather for outdoor activities.")
else:
    print("Stay indoors.")
Output
Good weather for outdoor activities.
1
temperature > 30→True(35 > 30)
2
not is_raining→True(not False = True)
3
True and True→True→ outdoor!
The condition combines and with not. Python evaluates not first (higher precedence), then and.
💡 Reading it in English
"If temperature is above 30 AND it is NOT raining" — maps directly to the Python code. Logical operators often match everyday language.
🎯 Try It
Change is_raining = True. Now not True = False → True and False = False → "Stay indoors."

Change temperature = 25. What happens?

Nested if Statements
An if statement placed inside another if statement
06 · Nested if
What Is a Nested if?
📖 Definition
A nested if is an if statement written inside another if statement. The inner check only runs if the outer check passes.
Structure
if outer_condition:
# outer block runs
if inner_condition:
print("Both true!")
else:
print("Outer true, inner false")
else:
print("Outer condition failed")
🎬 Analogy: Cinema Booking
First, you check whether a ticket is available.
If yes, then you check whether you can afford it.

The second check only matters if the first check passes — that's nesting!
⚠️ When to Use Nested vs and
Use nested when the second decision only makes sense after the first passes.

Use and when both conditions are independent and you simply need both to be true.
06 · Nested if
Example 7 — Cinema Ticket
Python
ticket_available = True
can_afford = True

if ticket_available:
    if can_afford:
        print("Enjoy the movie!")
    else:
        print("You cannot afford the ticket.")
else:
    print("Sorry, no tickets are available.")
Output
Enjoy the movie!
How the Decision Flows
OUTER CHECK
ticket_available → True → enter outer block
↓
INNER CHECK
can_afford → True → "Enjoy the movie!"
🎯 Try It
Change can_afford = False → see the inner else.
Change ticket_available = False → the inner if is never reached!
06 · Nested if
Example 8 — Grade Classification with Nesting
Python
score = 72

if score >= 50:
    print("You have passed.")
    if score >= 90:
        print("Grade: A")
    elif score >= 75:
        print("Grade: B")
    else:
        print("Grade: C")
else:
    print("You have failed.")
Output (score = 72)
You have passed.
Grade: C
Tracing score = 72
Outer: 72 >= 50 → True → "You have passed."
↓ enter inner block
72 >= 90 → False
72 >= 75 → False
else → "Grade: C" ✓
🎯 Try It
Change to score = 91 → "Passed + Grade: A"
Change to score = 45 → "You have failed." (inner block never runs)
07 · Best Practices
Keeping Conditions Simple and Readable
❌ Hard to Read — One Long Condition
Python
if age >= 18 and age <= 65 and is_member and not is_banned and score > 50:
    print("Eligible.")
✅ Clear — Named Boolean Variables
Python
is_valid_age = 18 <= age <= 65
is_eligible_member = is_member and not is_banned
has_passed = score > 50

if is_valid_age and is_eligible_member and has_passed:
    print("Eligible.")
Why the Second Version Is Better
📖
Reads Like English
The final if reads almost like a sentence.
🐛
Easier to Debug
If there is an error, you can print each variable to find it.
🔧
Easier to Modify
Change one piece without touching the others.
👥
Team Friendly
Other people can understand your code quickly.
💡 Rule of Thumb
If a condition needs more than 2–3 parts, split it into named Boolean variables first.
08
Common Mistakes
Five errors beginners make — and how to fix each one
08 · Common Mistakes
Mistakes 1 & 2
❌ Mistake 1: Using and When or Is Needed
A variable cannot equal two different values at the same time. day == "Sat" and day == "Sun" is always False.
❌ Always False
Logic Error
if day == "Saturday" and day == "Sunday":
    print("Weekend")
✅ Correct
Fixed
if day == "Saturday" or day == "Sunday":
    print("Weekend")
❌ Mistake 2: Forgetting to Repeat the Variable
"blue" alone is a non-empty string — Python treats it as always True! Always write the full comparison for each value.
❌ Always True!
Logic Error
if colour == "red" or "blue":
    print("Match")
✅ Correct
Fixed
if colour == "red" or colour == "blue":
    print("Match")
08 · Common Mistakes
Mistakes 3, 4 & 5
❌ Mistake 3: Misunderstanding not
Avoid not variable == True — it adds unnecessary complexity. Use not variable directly.
❌ Complex
Unclear
if not is_open == True:
    print("Closed")
✅ Clear
Fixed
if not is_open:
    print("Closed")
❌ Mistake 5: Incomplete Comparisons with and
Each condition must be a complete comparison. and "moderator" is always True (non-empty string), not a comparison!
❌ Wrong
Always True
if not role == "admin" and "moderator":
    print("Limited")
✅ Fixed
Fixed
if role != "admin" and role != "moderator":
    print("Limited")
❌ Mistake 4: Incorrect Indentation in Nested if
Every block inside an if must be consistently indented. Missing indentation causes an IndentationError.
❌ Wrong — IndentationError
IndentationError
if has_ticket:
print("Ticket confirmed.")  # missing!
    if is_adult:
        print("Adult entry.")
✅ Fixed
Python
if has_ticket:
    print("Ticket confirmed.")
    if is_adult:
        print("Adult entry.")
Summary
Lecture 5 — Key Concepts
  • and — returns True only when both conditions are True
  • or — returns True when at least one condition is True
  • not — reverses a Boolean: True → False, False → True
  • Precedence: not first, then and, then or
  • Nested if — inner check runs only when outer check passes
  • Always write complete comparisons: x == "a" or x == "b"
  • For complex conditions, use named Boolean variables for clarity
Quick Reference
Python
# and: both must be true
if age >= 18 and has_id:
    print("Entry allowed.")

# or: at least one must be true
if is_student or is_senior:
    print("Discount applied.")

# not: reverses the condition
if not is_logged_in:
    print("Please log in.")

# nested if
if has_ticket:
    if seat_available:
        print("Enjoy the show!")
